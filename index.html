Case snapshot

What this looks like in practice

I work across product, ML, trust systems, and go-to-market. The through-line is execution: take a messy problem, build the system, and make it useful in the real world.

Kampd

Discovery, trust, and ranking systems for a fast-growing community product

I built ML-powered discovery, trust, and personalization systems across an early-stage community platform with a rapidly growing user base. That meant owning ranking, experimentation, and applied ML pipelines end-to-end rather than handing off strategy and disappearing.

What shipped

Core systems

Trust layer system 0–100 real-time scoring
Feed ranking system +25% activation
Auto-moderation -40% manual review

Trust layer system

Built real-time trust scoring (0–100) combining behavioral signals, content quality, and model confidence.

Used those scores to influence ranking and moderation decisions in production.

Feed ranking system

Built ML ranking pipelines with Python and APIs using embeddings, relevance tuning, and content quality signals.

Improved activation by 25% by making discovery more relevant earlier in the user journey.

Auto-moderation

Built policy-driven moderation workflows on top of trust scoring and behavioral signals.

Reduced manual moderation review by 40% while preserving quality and trust.
